Angular Forms

Template-Driven vs Reactive Forms

1. Module Imports

Template-Driven

import { FormsModule } from '@angular/forms'; @Component({ imports: [FormsModule] })
  • Uses FormsModule
  • Enables ngModel directive
  • Template-based approach

Reactive

import { ReactiveFormsModule } from '@angular/forms'; @Component({ imports: [ReactiveFormsModule] })
  • Uses ReactiveFormsModule
  • Enables FormBuilder & FormGroup
  • Component-based approach

2. Component Setup

Template-Driven

export class TemplateFormComponent { user = { name: '', email: '', age: null, password: '' }; }
  • Simple data model object
  • No form controls setup needed
  • Direct property binding

Reactive

export class ReactiveFormComponent { userForm: FormGroup; constructor(private fb: FormBuilder) { this.userForm = this.fb.group({ name: ['', [Validators.required]], email: ['', [Validators.email]], age: ['', [Validators.min(18)]] }); } }
  • Explicit FormGroup creation
  • FormBuilder for setup
  • Validators defined in component

3. HTML Template Binding

Template-Driven

<form #userForm="ngForm"> <input type="text" name="name" [(ngModel)]="user.name" #name="ngModel" required> <div *ngIf="name.invalid && name.touched"> Name is required </div> </form>
  • Two-way binding with [(ngModel)]
  • Template reference variables
  • HTML validation attributes

Reactive

<form [formGroup]="userForm"> <input type="text" formControlName="name"> <div *ngIf="getErrorMessage('name')"> {{ getErrorMessage('name') }} </div> </form>
  • FormGroup binding
  • formControlName directive
  • Component methods for validation

4. Validation Approach

Template-Driven

// Validation in template <input required minlength="2" email> // Access validation state <div *ngIf="name.errors?.['required']"> Name is required </div>
  • HTML validation attributes
  • Template-based error checking
  • Limited custom validation

Reactive

// Validation in component this.userForm = this.fb.group({ name: ['', [ Validators.required, Validators.minLength(2) ]], email: ['', [Validators.email]] }); // Custom validation method getErrorMessage(fieldName: string) { const control = this.userForm.get(fieldName); return control?.errors ? 'Error message' : ''; }
  • Component-based validators
  • Easy custom validation
  • Programmatic error handling

5. Form Submission

Template-Driven

onSubmit(form: any) { if (form.valid) { console.log('Form data:', this.user); } }
// Template <form (ngSubmit)="onSubmit(userForm)"> <button [disabled]="userForm.invalid"> Submit </button> </form>
  • Pass form reference to method
  • Access data via component property
  • Form validation via template reference

Reactive

onSubmit() { if (this.userForm.valid) { console.log('Form data:', this.userForm.value); } }
// Template <form [formGroup]="userForm" (ngSubmit)="onSubmit()"> <button [disabled]="userForm.invalid"> Submit </button> </form>
  • Direct FormGroup access
  • Form value via .value property
  • Built-in validation state

6. Custom Validation

Template-Driven

// Limited custom validation // Requires custom directive @Directive({ selector: '[customValidator]' }) export class CustomValidatorDirective { // Complex setup required }
  • Requires custom directives
  • More complex implementation
  • Limited flexibility

Reactive

// Easy custom validation passwordMatchValidator(form: FormGroup) { const password = form.get('password'); const confirmPassword = form.get('confirmPassword'); if (password?.value !== confirmPassword?.value) { confirmPassword?.setErrors({ passwordMismatch: true }); return { passwordMismatch: true }; } return null; } // Apply to form this.userForm = this.fb.group({...}, { validators: this.passwordMatchValidator });
  • Simple function-based validators
  • Cross-field validation easy
  • Highly flexible and testable

When to Use Which?

✅ Template-Driven Forms

  • Simple forms with basic validation
  • Rapid prototyping
  • Forms that mirror data model closely
  • Team prefers declarative approach
  • Contact forms, registration forms

✅ Reactive Forms

  • Complex forms with custom validation
  • Dynamic forms that change at runtime
  • Extensive unit testing needed
  • Cross-field validation requirements
  • Multi-step wizards, business forms

Summary

Template-Driven

  • FormsModule + ngModel
  • HTML-based validation
  • Automatic form controls
  • Simple setup
  • Best for basic forms

Reactive

  • ReactiveFormsModule + FormBuilder
  • Component-based validation
  • Explicit form controls
  • More setup required
  • Best for complex forms

Choose based on your form complexity and requirements!
